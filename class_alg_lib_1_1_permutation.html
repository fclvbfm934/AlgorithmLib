Detailed Description

The Permutation class, which stores a Permutation object. The Permutation object can act like a functor, which can permute the elements of a container.

Permutation AlgLib::Permutation::next ( bool &  is_largest) const
virtual

Gives the next Permutation that is lexicographically larger. What this means is that given the Permutation (1, 2, 0), this will be larger than (1, 0, 2) because going from left to right and comparing, the 2 in the second position is bigger than the 0.

Parameters
is_largestA boolean storage variable to see whether or not the Permutation was already the largest one (where the elements will be in reverse order) When the largest Permutation has been reached, the identity Permutation (which is the smallest out of all Permutations) will be returned and is_largest will be set to true. Otherwise, is_largest will be set to false.
Returns
The smallest Permutation object that is lexicographically larger.

T AlgLib::Permutation::operator() ( const T &  container,
int  start,
int  last 
) const

Permutes the elements of container[start...(last-1)]. Note that last is not included in the permutation.

This functor will rearrange (or permute) the elements of the container. So, a Permutation of (2, 1, 0, 4, 3) will mean that container[start + 0] is mapped to container[start + 2]. This also means that container[start + 4] and container[start + 3] will be flipped.

Template Parameters
TThe type of the container. So, if the container is an std::vector<int>, then T would be std::vector<int>
Parameters
startWhere to start the permuting. Defaults to 0.
lastThe index after where the permutation stops. Thus, container[last] is not in the permutation. Defaults to container.size().
Returns
Return the permuted container. Note that the container itself was not modified.
Exceptions
std::length_errorThrows this exception if the size of the container (or last - start) is not the same as the Permutation length.

Permutation AlgLib::Permutation::operator* ( const Permutation &  other) const
virtual

The multiplication operator is overloaded to act like Permutation composition.

Usage would be something like: Permutation perm1({3, 2, 1}); Permutation perm2({2, 1, 3}); Permutation perm3 = perm1 * perm2;

This will result in a Permutation of (2, 3, 1).

Returns
The Permutation object that is the composition of the two Permutation objects
Exceptions
std::length_errorWhen the Permutations do not have the same size.
